## 3. 日期跨度与舍入

令连续住宿总晚数为 `N`，`P1/P7/P14/P30` 是所选价格产品的基准总价：

| 总晚数 | 档位 | 精确总价 |
|---:|---:|---|
| `1 <= N < 7` | 1 夜 | `N × P1` |
| `7 <= N < 14` | 7 夜 | `N × P7 / 7` |
| `14 <= N < 30` | 14 夜 | `N × P14 / 14` |
| `N >= 30` | 30 夜 | `N × P30 / 30` |

- “临住”严格小于 7 夜；6 夜属于临住，7 夜进入 7 夜档。
- 先以有理数计算完整住宿总价，最后只舍入一次到人民币整数元。
- 正数 `.5` 使用 half-up，不使用银行家舍入，不逐晚或逐分段舍入。
- 跨自然月不拆单；只看完整连续区间的总晚数。
- 允许 6→7、13→14、29→30 夜出现“多住一晚总价下降”；不平滑、不保底、不沿用上一档。
- `TRANSIENT/WEEKLY/MONTHLY/CUSTOM/FIXED_TERM/ROLLING` 是经营标签，不是六套公式。正常现金住宿由连续日期区间和锁定产品决定档位。

## 4. 连续 Stay 边界

MVP 中累计周期是一个 Order 的同一 Stay 内 `[originalArrival, finalDeparture)`：

- `EXTEND_STAY` 从原始入住日累计到新退房日，不只计算新增晚数。
- `SHORTEN_STAY` 按缩短后的完整区间重新选档，不保留缩短前档位。
- 跨月不会打断累计。
- 同一 Stay 内换房分段若前段 `departureDate` 等于后段 `arrivalDate`，日期无缝且累计不断档。
- 已 `CHECK_OUT` 的订单不会与另一个订单自动合并，也不新增 `continuationOfOrderId`。至少空出一个服务日后重新入住，是新的 Order/Stay 与新的计价周期，从新入住日重新计算。

这个边界保持订单资金事实不跨订单分摊；需要保持连续累计时，运营必须在原 Stay 内续住或形成无缝换房分段。

## 5. 换房

换房后的统一档位由整个 Stay 的累计总晚数 `N` 决定一次。对每个分段使用同一档位下该分段价格产品的日分数：

`segmentNights × productAnchorAmount / anchorNights`

所有分段精确金额相加后才对完整 Stay 总价 half-up 舍入一次：

- 同价格产品之间换房金额不变。
- 跨价格产品按各分段产品锚点形成新总价。
- 床位到多人间整间组合也使用同一累计档位。
- 换房、续住和缩短继续使用成交时锁定的同一个 policy version，并追加 amendment 与 pricing revision。

## 6. 会员、免费与手工指定

### 会员住宿

- 先按具体服务日期形成 `coverageSet`，且 ROOM_NIGHT/BED_NIGHT 必须匹配库存单位。
- 已选择会员但可用权益为 0 时，仍属于会员计价；所有未覆盖日期逐日使用锁定产品的 `P1`。
- 部分覆盖时，每个未覆盖日期使用 `P1`；未覆盖晚数即使累计达到 7/14/30，也不切换跨度档。
- 新权益 Lot 可在最终履约结算前覆盖尚未覆盖日期；同一日期不可重复覆盖或重复核销。
- 预订确认冻结，成功入住时把相应 HELD 权益转为 CONSUMED；入住前取消或未到释放 HELD。普通取消、缩短不得恢复已经 CONSUMED 的权益。
- 会员合同购买款不可退款；释放住宿权益不等于退会员费。

### 免费住宿

- 免费住宿与会员住宿是两套业务：`currentContractAmount=0`，不冻结、释放或核销会员权益。
- 创建必须保存免费原因；延住、缩短和取消必须保存本次变更原因。
- 日期、房型或状态变化继续追加 amendment、零金额 pricing revision 和审计，不需要 MVP 审批流或批准人。

### 手工指定最终总价

- 操作员输入本 revision 的最终 `currentContractAmount`，必须是非负人民币整数元，并填写原因。
- 系统先保留可复算政策基础报价，再保存 `manualAdjustment = targetFinalAmount - policyBaseAmount`。
- Preview、Receipt 与 Query 必须同时追溯政策基础价、指定最终价和差额。
- 后续续住、缩短或换房产生的新 revision 不继承旧 adjustment；除非操作员再次明确指定最终价。
- 指定最终价不是收款、退款、到账、已结清或会计收入。

## 7. 资金差额边界

PMS 只复算：

- `currentContractAmount`
- `netRecordedCollection`
- `collectionDifference = currentContractAmount - netRecordedCollection`

缩短或会员续充后，人工退款提示只取 `max(0, netRecordedCollection - currentContractAmount)`。退款仍须由工作人员发起，引用原 COLLECTION，并录入退款自身真实 `transactionReference`；系统不连接微信、银行或支付平台，也不自动退款或核销。
